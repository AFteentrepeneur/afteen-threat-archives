# ÄFteeñ Threat Archives — Research Repository

**Public research notebook, source archive, and methodology log for the [AFteen Threat Archives](https://youtube.com/@AFteenThreatArchives) YouTube channel.**

This repo documents the investigations behind every video: sources, timelines, methodology, and defensive lessons. It's the media arm's public research trail — nothing here is exclusive to the channel, and everything is built from publicly available information.

Produced by [AFteen SecLabs](#) as part of our mission to make cybercrime patterns understandable and defensible.

---

## What This Is

Each video we publish follows an investigator's-report structure: **Evidence → Timeline → Techniques → Patterns → Defensive Lessons**. This repo is where that research lives in full — more detail than fits in a 12–18 minute video, with citations you can check yourself.

## What This Is Not

- Not a how-to guide for offensive techniques
- Not a source of working exploits, malware samples, or credentials
- Not a directory of live dark web marketplaces or leak sites
- Not legal advice

See [DISCLAIMER.md](./DISCLAIMER.md) for the full policy.

## Repository Structure

| Folder | Purpose |
|---|---|
| `/cases/` | One file per major investigation (e.g., `lockbit.md`, `colonial-pipeline.md`) — timeline, sources, patterns, defensive takeaways |
| `/dark-web/` | Marketplace analyses, takedown writeups, dark web economy overviews |
| `/patterns/` | Thematic/recurring attacker behavior analysis (phishing evolution, ransomware rebranding, etc.) |
| `/resources/` | Curated links to public reports, tools, and reference material, with notes |
| `/methodology/` | How we research, verify, and write every case — see [methodology/methodology.md](./methodology/methodology.md) |
| `/VIDEO-SCRIPTS/` | Script templates and structure (not full scripts — see note below) |
| `/ideas/` | Backlog of case/topic ideas under consideration |
| `ROADMAP.md` | What's coming next |

> Note: `/VIDEO-SCRIPTS/` holds structural templates, not verbatim scripts for every video — final voiceover scripts stay production-side.

## How Each Case File Is Built

Every entry in `/cases/` follows the same template — see [methodology/case-template.md](./methodology/case-template.md). This keeps our research consistent and lets you cross-check our sources directly.

## Sources We Draw From

Public reporting only: law enforcement and government advisories, court documents, established security research (e.g., Krebs on Security), vendor threat reports, and journalist investigations. Every claim in a case file is sourced — see the "Sources" section at the bottom of each one.
